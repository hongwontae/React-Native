1. app.json
=> 앱의 주요 설정 파일이다.
=> name, slug(프로젝트 고유 식별자), version, orientation(앱의 가로, 세로 설정), icon, splash(초기 화면)....


2. babel.config.js
=> 최신 JS 문법 및 JSX를 하위 호환성 있게 트랜스파일링하기 위한 설정 파일


3. .expo
=> EXPO가 사용하는 내부 설정과 캐시 파일들이 들어 있는 폴더


4. 요소가 height을 명시적으로 설정하지 않으면 전체 높이를 차지한다.


5. RN에서는 JSX 문법을 사용하고 각 태그들(컴포넌트)을 네이티브 컴포넌트라고 부른다.


6. flex : column일 떄 height을 명시적으로 주지 않았을 떄
=> 넓이는 가용할 전체 넓이를 받아들이고 높이는 컨텐츠 크기만큼 갖는다.
=> 컨텐츠 크기라는 것은 패딩을 합친 값이다.
=> 원래는 height을 100%로 갖는데 flex : column이라 컨텐츠 높이만큼만 갖는것이다. flex : 1를 주게 되면 100%를
사용한다.


7. View는 가용할 공간을 전부 사용한다.
=> 가용할 공간이라는 것은 컨텐츠 크기이다.
=> 그래서 flex : 1, flex : column을 통해 background 작업을 한다.
