1. Styling
1-1. pressable의 ios pressed func은 android에게도 영향을 미친다.
1-2. ios의 boxShadow는 elevation의 영향을 받지 않는다. android는 모든 box-shadow의 영향을 받는다.
1-3. components의 style prop은 함수를 받을 수 있고 그 함수는 실행된 상태이다. 다만 매개변수 pressed의 이벤트를
트리거하는 효과가 있을 뿐이다.


2. React Navigation
2-1. React Navigation은 컴포넌트 기반 라이브러리이다.

2-2. NavigationContainer => navigation의 기능을 사용할 모든 컴포넌트들을 덮는 컴포넌트

2-3. const Stack = createNativeStackNavigator() 
=> 함수를 호출하면 3개의 navi 기능을 도와주는 프로퍼티가 할당된다.

2-4. <Stack.Navigator><Stack.Navigator/>
=> <Stack.Screen>을 덮는 컴포넌트

2-5. <Stack.scrren>
=> navi 기능을 사용할 컴포넌트를 등록하는 컴포넌트
=> name : 등록할 컴포넌트의 별칭
=> component : 등록할 컴포넌트


3. 기본 설정
